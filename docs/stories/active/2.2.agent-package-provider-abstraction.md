## User Story

**As a** developer da equipa Expressia construindo o pipeline 3-stage do Cérebro AI,
**I want** um package `@meu-jarvis/agent` com abstracção de providers LLM (Anthropic Sonnet primary, OpenAI fallback flag-gated) que encapsula retry/backoff, circuit breaker, error mapping, prompt caching, OTel tracing e PII redaction,
**so that** as Stories 2.4 (Classifier), 2.5 (Planner+Executor) e 2.10 (Benchmark) consomem providers via uma única interface tipada — sem repetir lógica de retry, sem fugas de PII para logs, e com hot-swap entre providers controlável por feature flag (NFR21).

### Contrato `ProviderInterface`

- **AC2:** `packages/agent/src/providers/interface.ts` exporta `ProviderInterface` tipada com Zod:
  ```ts
  export interface ProviderInterface {
    readonly id: 'anthropic' | 'openai';
    readonly model: LlmModel; // de packages/db agent_run_status enum (claude-sonnet-4-5 / gpt-4o-mini)
    complete(input: ProviderCompleteInput): Promise<ProviderCompleteOutput>;
  }
  ```
  Schemas Zod em `contracts.ts`:
  - `ProviderCompleteInputSchema`: `{ system: string, messages: Array<{role, content}>, tools?: ToolDefinition[], cacheControl?: 'ephemeral'|null, temperature?: number, maxTokens?: number, traceId: string, householdId: string }`
  - `ProviderCompleteOutputSchema`: `{ provider: 'anthropic'|'openai', model: LlmModel, content: string|null, toolCalls: Array<{name, input}>, finishReason: 'stop'|'tool_use'|'length'|'error', tokensInput: number, tokensOutput: number, costEur: number, latencyMs: number, cacheHit: boolean }`
  - Todos os campos PT-PT-friendly (mensagens de erro em português) quando aplicável.
  - `LlmModel` é tipo derivado do enum `llm_model` exportado de `@meu-jarvis/db` (single source of truth — Article IV).

### Adaptador Anthropic (provider primary)

- **AC3:** `packages/agent/src/providers/anthropic.ts` implementa `ProviderInterface`:
  - Usa SDK oficial `@anthropic-ai/sdk` (a adicionar como dep do package)
  - Lê `ANTHROPIC_API_KEY` de env (sem hardcoding; sem fallback). Se ausente em runtime, lança `MissingApiKeyError` (definida em `errors.ts`).
  - Suporta **Anthropic prompt caching** (`cache_control: ephemeral`) no system prompt + tool definitions JSON. Aplicado quando `cacheControl === 'ephemeral'` no input. Cost router (Story 2.9) controla activação; este adaptador apenas implementa.
  - Calcula `costEur` a partir de `tokensInput`/`tokensOutput` × tarifa Sonnet vigente (constantes em `pricing.ts` — `CLAUDE_SONNET_INPUT_USD_PER_1M`, `_OUTPUT_USD_PER_1M`, `_CACHE_READ_USD_PER_1M`, `_CACHE_WRITE_USD_PER_1M`; conversão USD→EUR usa rate fixo `0.92` parametrizável via env `AGENT_USD_TO_EUR_RATE` — Story 2.9 refinará com FX live).
  - Mapeia `cacheHit: true` quando a resposta inclui `usage.cache_read_input_tokens > 0`.
  - Modelo default `claude-sonnet-4-5` (constante exportada `CLAUDE_SONNET_DEFAULT`).

### Adaptador OpenAI (fallback flag-gated)

- **AC4:** `packages/agent/src/providers/openai.ts` implementa `ProviderInterface`:
  - Usa SDK oficial `openai` (a adicionar como dep do package)
  - Lê `OPENAI_API_KEY` de env. Ausência lança `MissingApiKeyError`.
  - **Flag-gated:** factory `getProvider({ preferredProvider })` em `providers/index.ts` selecciona Anthropic por default. OpenAI só é seleccionado quando:
    1. `preferredProvider === 'openai'` explicitamente (caso Story 2.4 Classifier que requer GPT-4o-mini), OU
    2. Feature flag `AGENT_FALLBACK_OPENAI_ENABLED === 'true'` (env) **E** Anthropic circuit breaker está aberto.
  - Modelo default `gpt-4o-mini` (constante `OPENAI_GPT4O_MINI_DEFAULT`).
  - Calcula `costEur` análogo (constantes `GPT4O_MINI_INPUT_USD_PER_1M`, `_OUTPUT_USD_PER_1M`).
  - **NÃO** suporta prompt caching (OpenAI não tem equivalente directo no MVP — `cacheControl` é silenciosamente ignorado, `cacheHit: false` sempre).

### Retry, backoff e circuit breaker

- **AC5:** `packages/agent/src/retry.ts` exporta `withRetry(fn, opts)`:
  - Backoff exponencial: `delayMs = baseMs * 2^attempt + jitterMs` com `baseMs=200`, jitter `±50ms`
  - Max attempts default `3` (configurável via opts)
  - Retry **APENAS** para erros idempotentes-safe: `RateLimitError` (HTTP 429), `TimeoutError`, `ServerError` (HTTP 5xx), `NetworkError`. **NÃO retry** para `AuthError` (401/403), `BadRequestError` (400/422), `ContentPolicyError`.
  - `Retry-After` header (Anthropic + OpenAI) substitui o backoff exponencial quando presente.
  - Cada retry incrementa span attribute `agent.provider.retry_count`.

- **AC6:** `packages/agent/src/circuit-breaker.ts` exporta `CircuitBreaker` class por provider id:
  - Estados: `closed`, `open`, `half_open`
  - Trip threshold: 5 falhas consecutivas (não-retryable OU retry esgotado) em janela de 60s → estado `open` durante 30s
  - `half_open` após 30s permite 1 probe; sucesso → `closed`, falha → `open` por mais 60s
  - Em `open`, chamadas levantam `CircuitOpenError` imediatamente (fast-fail).
  - Estado é **per-process in-memory** (não distribuído — cada Vercel function instance tem o seu). Cluster-aware fica para Story 2.9 (Upstash).

### Error mapping

- **AC7:** `packages/agent/src/errors.ts` define hierarquia tipada:
  ```
  ProviderError (abstract)
   ├─ RateLimitError(retryAfterMs?)
   ├─ TimeoutError(timeoutMs)
   ├─ ServerError(httpStatus, body?)
   ├─ NetworkError(cause)
   ├─ AuthError(httpStatus)
   ├─ BadRequestError(httpStatus, message)
   ├─ ContentPolicyError(reason)
   ├─ MissingApiKeyError(providerId)
   └─ CircuitOpenError(providerId, retryAfterMs)
  ```
  - Todos os errors PT-PT (`message` e `userMessage` campos; `userMessage` é o texto a mostrar ao utilizador final via Story 2.6 endpoint, neutro de implementação).
  - Mapping de SDK errors → `ProviderError` em `providers/anthropic.ts` e `openai.ts` via switch sobre `error.status`/`error.code`/`error.name`.
  - **Nenhum** erro carrega o prompt original no `message` (PII).

### Tracing OTel + PII redaction

- **AC8:** `packages/agent/src/tracing.ts` reutiliza `@meu-jarvis/observability`:
  - Função `withProviderSpan(providerId, model, fn)` cria span `agent.provider.call` com attributes:
    - `agent.provider` (`anthropic`|`openai`)
    - `agent.model` (string)
    - `agent.tokens_input`, `agent.tokens_output` (post-call)
    - `agent.cost_eur` (post-call)
    - `agent.latency_ms` (post-call)
    - `agent.cache_hit` (boolean)
    - `agent.retry_count` (number)
    - `agent.fallback_used` (boolean — true quando OpenAI seleccionado por circuit open Anthropic)
    - `agent.trace_id` (do input)
  - **NUNCA** adicionar prompt content, tool inputs/outputs, ou messages como attributes (PII).
  - `recordSpanError` chamado em qualquer `ProviderError` que escape do retry. Span status `OK` em sucesso.

- **AC9:** `packages/agent/src/redaction.ts` complementa `PII_REDACT_PATHS` do logger:
  - Helper `redactProviderPayload(payload)` retorna shallow copy sem campos: `system` (pode conter PII em prompts customizados de tools), `messages[].content`, `tools[].input_schema` (não PII mas verboso).
  - Usado **antes de qualquer log/Sentry capture** dentro do package agent. Logs internos do package usam `logger.debug({ provider, model, traceId, householdHash })` — **nunca** prompt/messages.
  - `householdHash = hashForCorrelation(householdId)` (não logar `household_id` raw em providers — OK em endpoints superiores onde já passou pela camada de auth).
