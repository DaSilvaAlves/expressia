## Story

**As a** equipa de produto da Expressia,
**I want** trocar o modelo default do Executor de `claude-sonnet-4-5` para `claude-haiku-4-5-20251001`,
**so that** o custo de regime do cérebro AI baixe de forma significativa sem quebrar tool calling, prompt caching (`cache_control: ephemeral`) nem a telemetria de custo existente.

---

## Contexto e Decisão

**Decisão tomada pelo Eurico em 2026-05-30.**

Antecedente imediato: ADR-001 deu NO-GO ao OpenRouter (data residency UE só em plano enterprise, fora da política NFR16/NFR17 do projecto) — ficam chaves directas OpenAI + Anthropic. O Classifier mantém-se `gpt-4o-mini` (OpenAI). Só o Executor muda de modelo.

Motivação: o modelo `claude-sonnet-4-5` tem pricing superior ao necessário para o volume de regime actual. O Haiku 4.5 (`claude-haiku-4-5-20251001`) suporta **tool calling** e **prompt caching nativo Anthropic** (`cache_control: ephemeral`, conforme NFR11), pelo que a mudança é low-risk funcionalmente.

O medo de "factura disparar" é mitigado em paralelo por spend caps no dashboard Anthropic (acção do Eurico, fora desta story). Esta story trata apenas da optimização no código.

Directiva estratégica activa (`refocus_core_before_billing`): billing/Stripe continua CONGELADO. Esta story serve o core (cérebro AI), não billing — está em scope.

**Modelo Sonnet permanece válido no enum e no override `opts.model`** — pode ser usado explicitamente. Só o *default* muda. Opus continua válido da mesma forma.

## Acceptance Criteria

1. O `AnthropicProvider` usa `claude-haiku-4-5-20251001` por default quando `opts.model` não é especificado; o override `opts.model` continua funcional e aceita Sonnet/Opus/Haiku.
2. A migration SQL adiciona `'claude-haiku-4-5'` ao enum Postgres `llm_model` via `ALTER TYPE … ADD VALUE IF NOT EXISTS`; a migration é idempotente e o `pnpm check:rls` (RLS Coverage Gate) continua verde.
3. `LLM_MODEL_VALUES` em `packages/agent/src/contracts.ts` inclui `'claude-haiku-4-5'`; `CLAUDE_HAIKU_DEFAULT` é exportado com valor `'claude-haiku-4-5-20251001'`; `CLAUDE_SONNET_DEFAULT` é mantido (outros callers não são quebrados).
4. `packages/agent/src/pricing.ts` contém constantes dedicadas para Haiku 4.5 (input, output, cache_read, cache_write em USD/1M tokens, com os valores oficiais Anthropic confirmados na implementação); a função `calculateAnthropicCost` (actualmente com 4 parâmetros de tokens, sem parâmetro `model`) é alterada para receber o modelo como argumento e seleccionar o conjunto de constantes correcto (Sonnet → constantes Sonnet; Haiku → constantes Haiku), sem reutilizar tarifas Sonnet para chamadas Haiku; o caller `providers/anthropic.ts:213` é actualizado em conformidade para passar o modelo.
5. As referências hardcoded `'claude-sonnet-4-5'` em `apps/web/src/app/api/agent/prompt/route.ts` e `apps/web/src/app/api/agent/prompt/[runId]/confirm/route.ts` são substituídas com distinção por tipo de uso: (a) `route.ts:481` (`executorModel` em `updateAfterPlanner`) e `confirm/route.ts:190` escrevem na **coluna `agent_runs.executor_model` (tipo enum `llm_model`)** — devem usar o short-form `'claude-haiku-4-5'` (o valor que existe no enum após AC2) e dependem da migration AC2 estar aplicada; (b) `route.ts:559` é um campo num **span OTel** (telemetria, string livre) — pode usar `'claude-haiku-4-5'` para coerência mas sem restrição de enum. Ambos os casos podem importar a constante `CLAUDE_HAIKU_DEFAULT` de `@meu-jarvis/agent` para evitar strings literais dispersas, desde que o `@dev` confirme que o valor da constante (`'claude-haiku-4-5-20251001'` ou `'claude-haiku-4-5'`) é o correcto para cada contexto.
6. A suite de testes passa limpa (`pnpm lint && pnpm typecheck && pnpm test && pnpm check:rls`): os testes que afirmavam `'claude-sonnet-4-5'` como default ou como único valor válido são actualizados para o novo estado; o teste `contracts.test.ts:38` (que rejeitava `'claude-haiku-3.5'`) é revisto para rejeitar apenas versões inválidas, não o Haiku 4.5 que agora é válido.
7. O `cache_control: ephemeral` continua a propagar-se nas chamadas Haiku (system prompt + última tool definition), idêntico ao comportamento actual com Sonnet — verificado pelos testes existentes de `providers/anthropic.test.ts`.

### Decisões de implementação

**[DEV-DECISION] D-2.12.1 — Duas constantes (enum short-form vs API ID).** AC3 exige `CLAUDE_HAIKU_DEFAULT = 'claude-haiku-4-5-20251001'` (API ID full-form), mas `LlmModel` (derivado de `LLM_MODEL_VALUES`) só contém o short-form `'claude-haiku-4-5'` (coerente com o enum Postgres, que guarda `'claude-sonnet-4-5'` sem sufixo). **[DEV-DECISION] D-2.12.3 — `getProvider` e Planner narrowing de `AnthropicModel`.** Ao tornar `AnthropicProvider.model` um `AnthropicModel` (= `Exclude<LlmModel,'gpt-4o-mini'>`), o boundary `getProvider` (que recebe `LlmModel`) e o `Planner` passaram a precisar de narrowing.
