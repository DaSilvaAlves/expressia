### Drizzle client pattern (Architecture §11.2, db-schema.md §7.3)

```typescript
// packages/db/src/client.ts
import { drizzle } from 'drizzle-orm/postgres-js'
import postgres from 'postgres'
import * as schema from './schema'

// Runtime queries — Supabase pooler (porta 6543)
const queryClient = postgres(process.env.DATABASE_URL!)
export const db = drizzle(queryClient, { schema })

// Para migrations — usar DATABASE_URL_DIRECT (porta 5432)
```

### Connection pooling (Architecture §11.2)

| Variável | URL format | Uso |
|----------|------------|-----|
| `DATABASE_URL` | `postgresql://...@db.xxx.supabase.co:6543/postgres` | Runtime queries (PgBouncer) |
| `DATABASE_URL_DIRECT` | `postgresql://...@db.xxx.supabase.co:5432/postgres` | Migrations (Drizzle Kit) |

Usar `DATABASE_URL_DIRECT` APENAS em `drizzle.config.ts`. Em produção (Vercel), apenas `DATABASE_URL` está disponível via variável de ambiente runtime.

### Supabase CLI (alternativa a Drizzle Kit push)

```bash
# Ligar ao projecto
supabase link --project-ref <REF>

# Aplicar migrations
supabase db push

# Verificar estado
supabase migration list
```
