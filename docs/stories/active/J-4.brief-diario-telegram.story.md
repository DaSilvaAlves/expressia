## Story

**As a** Eurico (único utilizador do Jarvis v1),
**I want** que o Jarvis me envie todas as manhãs às 07:30 (hora de Lisboa) uma mensagem no Telegram a resumir as minhas tarefas de hoje, as atrasadas e o estado das finanças do mês, escrita em PT-PT natural,
**so that** começo o dia com uma visão clara do que tenho de fazer sem ter de abrir a web app — e posso responder em conversa (o motor de J-2 já trata as acções).

## Contexto de âmbito

- Esta story entrega o brief com **tarefas + finanças** apenas. A **agenda (Google Calendar)** NÃO entra aqui — depende de J-3 (OAuth). Após J-3, um follow-up curto injecta os eventos do dia no mesmo brief.
- Reutiliza infra existente: Inngest (5 jobs já em prod), agregadores `lib/visao/queries.ts`, `telegram/client.ts` (`sendMessage`), e o provider OpenAI `gpt-4o-mini` (motor migrado em 24/06).

## Acceptance Criteria

1. Existe a tabela `daily_briefing_cache` (migration `0028`) com `id uuid PK`, `household_id uuid NOT NULL FK households ON DELETE CASCADE`, `briefing_date date NOT NULL` (dia em Europe/Lisbon), `message_text text NOT NULL`, `generated_at timestamptz NOT NULL`, `created_at timestamptz NOT NULL DEFAULT now()`, e constraint `UNIQUE (household_id, briefing_date)`. Aplicada via `pnpm db:migrate`.
2. `daily_briefing_cache` tem exactamente 4 RLS policies (SELECT/INSERT/UPDATE/DELETE) com predicate `household_id = public.current_household_id()`, colocadas em `0001_rls_policies.sql` via DO-block condicional (pattern D-J2.1 — `check-rls-coverage.ts:33` lê só a 0001). `pnpm check:rls` passa (29 tabelas). Schema Drizzle da tabela exportado.
3. Existe uma função Inngest `generateDailyBrief` com trigger `{ cron: 'TZ=Europe/Lisbon 30 7 * * *' }`, registada em `/api/inngest/route.ts`. (Inngest v3 suporta o prefixo `TZ=` — resolve DST nativamente.)
4. O job resolve os destinatários lendo `telegram_link` via `getServiceDb()` (identidade fora de sessão — uso legítimo SEC-10, igual a J-2). Para cada `{ chat_id, user_id, household_id }`, a agregação e a leitura/escrita de `daily_briefing_cache` correm dentro de `withHousehold({ userId, householdId }, fn)` — RLS viva (lição SEC-8.1).
5. A agregação reutiliza `getTasksToday`, `getTasksOverdue`, `getFinancesMonth`, `getAccountsBalance` de `@/lib/visao/queries` **sem alterar** essas funções.
6. A síntese do texto é gerada por LLM (`getProvider({ preferredProvider: 'openai' }).complete(...)`, `gpt-4o-mini`) com system prompt PT-PT, `temperature ≈ 0,6`, `maxTokens ≤ 320`. O texto é factual, conciso (≤ ~6 linhas), em português europeu, e cobre: tarefas de hoje (com contagem), tarefas atrasadas (com contagem), e finanças do mês (receita/despesa/saldo + saldo das contas). Se o LLM falhar, há um **fallback determinístico** (texto montado em código a partir dos mesmos dados) — o brief NUNCA fica por enviar por falha do LLM.
7. Idempotência por `{ household_id, briefing_date }`: antes de enviar, o job verifica `daily_briefing_cache` para o dia de hoje (Europe/Lisbon); se já existe, **não re-envia** (Inngest tem entrega at-least-once). Após envio com sucesso, grava a linha (`message_text` + `generated_at`).
8. Envio via `sendMessage({ chatId, text })`. Falha de envio é **não-fatal por household** (log + continue para os restantes; não bloqueia o job inteiro nem grava o cache desse household — para que possa ser re-tentado).
9. Logs com `childLogger`/`withSpan` e `hashForCorrelation` para `user_id`/`household_id`; **nunca** o texto do brief, títulos de tarefas ou valores financeiros em claro. Contam apenas contagens agregadas e hashes.
10. `pnpm lint`, `pnpm typecheck`, `pnpm --filter @meu-jarvis/web test` (com novos testes unitários), `pnpm build`, `pnpm check:rls` e `pnpm --filter @meu-jarvis/db-test test` passam sem erros.

## Dev Notes

### Decisões já fechadas (pré-implementação)

- **Timezone:** `{ cron: 'TZ=Europe/Lisbon 30 7 * * *' }` — Inngest v3 (`^3` no package.json) suporta o prefixo `TZ=`; resolve WET/WEST automaticamente. Sem ginástica de DST.
- **Idempotência:** tabela `daily_briefing_cache` (0028), unique `(household_id, briefing_date)`. `briefing_date` calculada em Europe/Lisbon.
- **Tenancy:** identidade via `telegram_link` (`getServiceDb`), agregação+cache via `withHousehold` (RLS viva). NÃO tocar `withHousehold`/`db-shim` (SEC-8 HOLD).
- **Síntese:** OpenAI `gpt-4o-mini` (provider de produção desde 24/06). Texto livre (sem tool calling). Fallback determinístico obrigatório.
