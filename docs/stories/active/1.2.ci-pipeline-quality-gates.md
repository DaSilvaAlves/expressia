## User Story

**As a** developer da equipa Expressia,
**I want** uma pipeline GitHub Actions que execute lint, typecheck, testes e o gate RLS de forma bloqueante em cada PR e push para main,
**so that** nenhum código com erros de qualidade ou que quebre a cobertura RLS pode ser mergeado, garantindo as NFRs de qualidade desde o primeiro commit.

## Dev Notes

### Estrutura do `ci.yaml` (Architecture §11.4)

```yaml
name: CI

on:
  pull_request:
  push:
    branches: [main]

jobs:
  quality:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
        with:
          version: 9
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'pnpm'
      - run: pnpm install --frozen-lockfile
      - run: pnpm lint
      - run: pnpm typecheck
      - run: pnpm test

  rls-gate:
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:16
        env:
          POSTGRES_USER: test
          POSTGRES_PASSWORD: test
          POSTGRES_DB: testdb
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5
        ports:
          - 5432:5432
    env:
      DATABASE_URL: postgresql://test:test@localhost:5432/testdb
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
        with:
          version: 9
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'pnpm'
      - run: pnpm install --frozen-lockfile
      - name: Apply migrations
        run: pnpm --filter db db:migrate
      - name: Check RLS coverage (NFR5)
        run: pnpm --filter db check-rls

  build:
    runs-on: ubuntu-latest
    needs: [quality]
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
        with:
          version: 9
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'pnpm'
      - run: pnpm install --frozen-lockfile
      - run: pnpm build
        env:
          NEXT_PUBLIC_SUPABASE_URL: ${{ secrets.NEXT_PUBLIC_SUPABASE_URL }}
          NEXT_PUBLIC_SUPABASE_ANON_KEY: ${{ secrets.NEXT_PUBLIC_SUPABASE_ANON_KEY }}
```

### Secrets necessários no GitHub (NFR8)

| Secret | Usado em | Necessário para |
|--------|----------|-----------------|
| `NEXT_PUBLIC_SUPABASE_URL` | job `build` | Build Next.js sem erros |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | job `build` | Build Next.js sem erros |

Os restantes secrets (ANTHROPIC_API_KEY, STRIPE_*) não são necessários em CI nesta story.

### Branch protection rules a documentar

- Require status checks before merging: `quality`, `rls-gate`, `build`
- Require branches to be up to date before merging
- Require at least 1 approval (opcional para MVP, recomendado)
- Restrict direct pushes to main

### Variáveis de ambiente do job `rls-gate`

O job `rls-gate` usa Postgres local (service container). Não precisa de credenciais Supabase reais — usa container temporário com schema migrado localmente. Esta separação é intencional: o gate RLS testa a estrutura SQL, não a instância Supabase.
