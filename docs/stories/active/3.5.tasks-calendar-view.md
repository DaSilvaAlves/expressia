## User Story

**Como** utilizadora da Expressia (`@household-owner` ou `@household-member`) que organiza tarefas com datas due no agregado familiar,
**Quero** uma vista calendário semanal em `/tarefas/calendario` com 7 colunas (Segunda a Domingo, PT-PT week start Monday), tarefas agrupadas pelo `due_date` no dia respectivo, drag-and-drop entre dias para reagendar (e drag de/para sidebar "Por agendar" para tarefas sem data), navegação semana anterior/próxima/hoje, e paridade mobile com vista 1-dia (swipe horizontal),
**Para que** possa planear a minha semana com referência visual ao tempo (vs lista cronológica ou Kanban por estado), arrastar tarefas entre dias para reorganizar sem abrir um modal, e capturar tarefas sem data num sidebar dedicado que persiste entre semanas — mantendo o mesmo padrão de drag-and-drop atómico e auditado que conheço da vista Kanban.

(c) Drag lifecycle (idêntico padrão Kanban Story 3.4):

(d) Drag PARA `UnscheduledSidebar`: PATCH body `{ due_date: null }` (remove data).

### 8. Week start Monday PT-PT + date-fns locale

**AC8:** Toda a lógica de datas usa **date-fns** com locale `pt`:

(a) Imports: `import { startOfWeek, addWeeks, format, parseISO } from 'date-fns'; import { pt } from 'date-fns/locale/pt';`

(b) `startOfWeek(date, { locale: pt, weekStartsOn: 1 })` — força Monday (R5 mitigation). NUNCA usar `getDay()` Date API com aritmética manual (depende do locale do servidor).

(c) Format de datas para UI: `format(date, 'd MMM', { locale: pt })` → "14 Mai" (não "14 May"). Format completo: `format(date, "d 'de' MMMM yyyy", { locale: pt })` → "14 de Maio 2026".

(d) Dia da semana abreviado: `format(date, 'EEEEEE', { locale: pt })` → "Seg, Ter, Qua, Qui, Sex, Sáb, Dom".

(e) ISO week format para URL state: `format(date, "yyyy-'W'II", { locale: pt })` → "2026-W21".

(f) **PO_FIX 2b v1.1:** Conversão `due_date` → dia visualizado: **NÃO precisa de `date-fns-tz` `utcToZonedTime`** — `tasks.due_date` é PG `date` type calendar-only (YYYY-MM-DD string sem hora nem fuso). Comparação directa `task.due_date === dayIso` (string vs string). `date-fns-tz` continua relevante apenas para campos `timestamp withTimezone` (`completed_at`, `created_at`, `updated_at`) caso o display precise de hora local — **NÃO para due_date**. Cross-confirmed em `packages/db/src/schema/tasks.ts:77` (`dueDate: date('due_date')` linha 77 — sem `withTimezone`, sem hora; campo separado `dueTime: text('due_time')` linha 79 para hora opcional HH:MM em texto).

**AC8 PASS:** Todos os formatos PT-PT correctos. Week start Monday garantido. DST handling via `date-fns-tz`. Sem regressões de fuso horário.

### Estrutura DndContext mínima (pattern Story 3.4)

```tsx
// WeekViewClient.tsx
'use client';

import { DndContext, DragOverlay, closestCorners } from '@dnd-kit/core';
import { useSensors, useSensor, PointerSensor, TouchSensor, KeyboardSensor } from '@dnd-kit/core';
import { sortableKeyboardCoordinates, SortableContext, verticalListSortingStrategy } from '@dnd-kit/sortable';

export function WeekViewClient({ initialTasks, unscheduledTasks, weekStart }: Props) {
  const [tasks, setTasks] = useOptimistic(initialTasks, optimisticReducer);
  const sensors = useSensors(
    useSensor(PointerSensor, { activationConstraint: { distance: 8 } }),
    useSensor(TouchSensor, { activationConstraint: { delay: 250, tolerance: 5 } }),
    useSensor(KeyboardSensor, { coordinateGetter: sortableKeyboardCoordinates }),
  );

  return (
    <DndContext
      sensors={sensors}
      collisionDetection={closestCorners}
      onDragStart={handleDragStart}
      onDragEnd={handleDragEnd}
      onDragCancel={handleDragCancel}
      accessibility={{
        announcements: announcementsCalendarPt,  // AC11 6 anúncios PT-PT
        screenReaderInstructions: {
          draggable: 'Premir espaço para mover entre dias. Setas para reposicionar. Enter para soltar. Esc para cancelar.',
        },
      }}
    >
      <UnscheduledSidebar tasks={unscheduledTasks} />
      <WeekView weekStart={weekStart} tasks={tasks} />
      <DragOverlay>
        {activeTask ? <CalendarTaskCard task={activeTask} mode="overlay" /> : null}
      </DragOverlay>
    </DndContext>
  );
}
```
