### AC5 — FUP-5.6.A: `isVisaoEmpty` optimizado com React `cache()` (perf)

**AC5:** A double-query na `/visao` é eliminada via `React.cache()` (DP-5.10.D = A). A heurística `isVisaoEmpty` e os widgets RSC deixam de fazer o mesmo round-trip à DB por page-load.

**(a)** As 6 funções de query em `apps/web/src/lib/visao/queries.ts` (`getTasksToday`, `getTasksOverdue`, `getFinancesMonth`, `getRecurrencesNext`, `getAccountsBalance`, `getCalendarWeek`) são envolvidas com `React.cache()`.

**(c)** Comportamento RLS preservado: `React.cache()` **NÃO** partilha entre requests de utilizadores diferentes (é per-request no RSC). O `getDb()` (authenticated, RLS via JWT) continua a ser usado em todas as queries — sem `getServiceDb()`.

**(d)** [DEV-DECISION] Se `React.cache()` levantar complicações (ex.: tipagem ou comportamento inesperado com o `getDb()` shim), alternativa **B** aceitável: partilhar os resultados como argumento explícito ao `<WidgetGrid>` (props drilling) — menos elegante mas igualmente correcto.

**AC5 PASS:** Grep positivo de `React.cache` ou nota de DEV-DECISION documentada com alternativa B. Comentário em `page.tsx` actualizado. `pnpm test` verde (zero regressão nas queries). `check:rls` EXIT 0 28 tabelas.

NOTA: o fix de segurança do auth gate (FUP-5.3.C) já foi entregue pela Story 5.0-hotfix (Done, 2026-05-29) — não é trabalho desta story.

**React `cache()` e RLS (FUP-5.6.A / DP-5.10.D).** `React.cache()` é scoped ao request — não partilha estado entre requests de utilizadores diferentes (sem cross-request leak). O `getDb()` que as queries usam é authenticated (RLS via JWT); o `cache()` apenas deduplica chamadas dentro do **mesmo** render tree de um request. Seguro para uso com multi-tenancy. Referência: Next.js 15 docs "Data Fetching: Caching".

**FUP-5.3.C — resolvido externamente.** O auth gate de `/tarefas` e `/financas` (`APP_PATH_PREFIXES`) foi corrigido pela Story 5.0-hotfix (Done, security gate PASS 9.5/10, 2026-05-29), com teste de cobertura anti-reincidência. Verificado: `middleware.ts` já inclui `/tarefas` e `/financas`. Não é trabalho desta story.

**FUP-5.3.D — root `/`.** O ficheiro `apps/web/src/app/page.tsx` actual é um componente estático que não verifica sessão. Converter para RSC com `async` + `getUser()` + `redirect()` (precedente `visao/page.tsx:141-146`) é a forma mais simples e consistente.
