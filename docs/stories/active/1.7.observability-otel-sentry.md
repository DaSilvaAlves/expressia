## User Story

**As a** developer/operator da Expressia,
**I want** telemetria OpenTelemetry configurada desde o primeiro endpoint, com Sentry para errors e Grafana Cloud para métricas/traces/logs,
**so that** a equipa pode observar latência, error rate e custo LLM desde o dia 1 — conforme NFR13 ("OTel em todos os services desde Epic 1, não afterthought").

## Dev Notes

### Stack de observabilidade (Architecture §9.1, ADR-004)

| Camada | Provider | Região | Razão |
|--------|----------|--------|-------|
| RUM / Web Vitals | Vercel Observability | UE | Built-in, zero overhead |
| Errors + Replay | Sentry (EU data residency) | UE | DX premium, PII redaction |
| Métricas + Traces + Logs | OTel SDK → Grafana Cloud (EU) | UE | Open standard, free tier suficiente até ~5k households |

**Grafana Cloud free tier:** 10k series de métricas + 50GB logs + 50GB traces — suficiente para o MVP.

### Logger Pino com redaction (Architecture §9.3, NFR12)

```typescript
// packages/observability/src/logger.ts
import pino from 'pino'

export const logger = pino({
  level: process.env.LOG_LEVEL || 'info',
  redact: {
    paths: [
      'email',
      'password',
      'nif',
      'iban',
      'prompt_text',
      '*.email',
      '*.password',
      'req.headers.authorization',
      'req.headers.cookie',
    ],
    censor: '[REDACTED]',
  },
  serializers: {
    err: pino.stdSerializers.err,
  },
})
```

**NUNCA** logar: conteúdo de prompts, email, NIF, IBAN, número de cartão, tokens de sessão.

### OTel instrumentation.ts (Next.js 15)

```typescript
// apps/web/instrumentation.ts
export async function register() {
  if (process.env.NEXT_RUNTIME === 'nodejs') {
    const { NodeSDK } = await import('@opentelemetry/sdk-node')
    const { OTLPTraceExporter } = await import('@opentelemetry/exporter-trace-otlp-http')
    const { getNodeAutoInstrumentations } = await import('@opentelemetry/auto-instrumentations-node')

    const sdk = new NodeSDK({
      serviceName: 'meu-jarvis-web',
      traceExporter: new OTLPTraceExporter({
        url: process.env.OTEL_EXPORTER_OTLP_ENDPOINT,
        headers: {
          Authorization: process.env.OTEL_EXPORTER_OTLP_HEADERS || '',
        },
      }),
      instrumentations: [
        getNodeAutoInstrumentations({
          '@opentelemetry/instrumentation-fs': { enabled: false }, // reduz ruído
        }),
      ],
    })

    sdk.start()
  }
}
```

### Sentry `beforeSend` hook (NFR12)

```typescript
// apps/web/sentry.server.config.ts
import * as Sentry from '@sentry/nextjs'

Sentry.init({
  dsn: process.env.SENTRY_DSN,
  environment: process.env.NODE_ENV,
  beforeSend(event) {
    // Remover PII antes de enviar
    if (event.request?.headers) {
      delete event.request.headers['authorization']
      delete event.request.headers['cookie']
    }
    if (event.user?.email) {
      event.user.email = undefined // NUNCA logar email em Sentry
    }
    return event
  },
  // Replay com máscara de texto (não captura conteúdo de inputs)
  integrations: [Sentry.replayIntegration({ maskAllText: true, blockAllMedia: true })],
  replaysSessionSampleRate: 0,    // desactivado por default
  replaysOnErrorSampleRate: 0.1,  // apenas em erros
})
```

### Métricas-chave (Architecture §9.2)

| Métrica | Source | Limiar de alerta |
|---------|--------|-----------------|
| `http.server.duration` (p95 `/api/me`) | OTel auto-instrumentation | > 200ms em 5min |
| `http.server.response_count` (5xx rate) | OTel | > 1% em 5min |
| `rls.policy_violations_total` | Log de RLS errors → Grafana | > 0 → CRITICAL |
| `db.client.operation.duration` p95 | OTel Postgres instrumentation | > 300ms sustained |

### Variáveis de ambiente necessárias (Architecture §12.3)

```bash
# Sentry
SENTRY_DSN=https://xxx@oXXX.ingest.de.sentry.io/YYY  # URL de.sentry.io = EU

OTEL_EXPORTER_OTLP_HEADERS=Authorization=Basic base64(instanceId:apiKey)
```

**Sentry EU:** O DSN deve conter `ingest.de.sentry.io` (Alemanha) ou `ingest.eu.sentry.io` — verificar que não é `ingest.sentry.io` (US).
