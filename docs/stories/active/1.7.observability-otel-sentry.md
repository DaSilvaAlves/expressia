### OTel instrumentation.ts via `@vercel/otel` (Next.js 15)

**Decisão arquitectural:** usar `@vercel/otel` em vez de `@opentelemetry/sdk-node` cru. Razão: `@vercel/otel` é mantido pelo Vercel, trata serverless cold-start automaticamente, e lê `OTEL_EXPORTER_OTLP_ENDPOINT` + `OTEL_EXPORTER_OTLP_HEADERS` directamente sem boilerplate.

```typescript
// apps/web/instrumentation.ts
import { registerOTel } from '@vercel/otel'

export function register() {
  registerOTel({
    serviceName: 'expressia-web',
    // OTLP endpoint + headers vêm de env vars automaticamente:
    // OTEL_EXPORTER_OTLP_ENDPOINT=https://otlp-gateway-prod-eu-west-6.grafana.net/otlp
    // OTEL_EXPORTER_OTLP_HEADERS=Authorization=Basic%20<base64(InstanceID:Token)>
  })
}
```

```typescript
// apps/web/next.config.ts (excerto)
import { withSentryConfig } from '@sentry/nextjs'

const nextConfig = {
  experimental: {
    instrumentationHook: true, // OBRIGATÓRIO para instrumentation.ts ser invocado
  },
  transpilePackages: ['@meu-jarvis/db', '@meu-jarvis/observability'],
  // ... resto da config
}

export default withSentryConfig(nextConfig, {
  org: process.env.SENTRY_ORG,           // 'eurico-xw'
  project: process.env.SENTRY_PROJECT,   // 'expressia-web'
  authToken: process.env.SENTRY_AUTH_TOKEN,
  silent: !process.env.CI,
})
```

**⚠️ Pontos de atenção (do handoff devops 2026-05-07):**

| Ponto | Detalhe |
|-------|---------|
| Base64 já formatado | `OTEL_EXPORTER_OTLP_HEADERS` já tem `Authorization=Basic%20<base64>` calculado pela UI Grafana — **NÃO recalcular nem decodificar**. SDKs OTel decodificam o `%20` (URL-encoded space) automaticamente. |
| `instrumentationHook` | Sem `experimental.instrumentationHook = true` em `next.config.ts`, o ficheiro `instrumentation.ts` é ignorado em build serverless. |
| Sourcemap upload Sentry | Build em CI precisa de `secrets.SENTRY_AUTH_TOKEN` no workflow GitHub Actions — sem este, o plugin Sentry emite warning não-bloqueante mas sourcemaps não são uploaded e stack traces ficam minificadas no Sentry. |
| Vercel free plan log limit | 1024 MB log per function execution — evitar `logger.info` verbose em hot paths produção (manter em `logger.debug` quando aplicável). |
| Sentry OTLP nativo (descoberto) | Sentry expõe agora endpoint OTLP nativo em `o4510848200278016.ingest.de.sentry.io/api/4510848213581904/integration/otlp`. Permitiria emitir traces directos para Sentry. **Decisão: NÃO usar nesta story** — manter dual-emission (Sentry para errors via SDK, Grafana para tudo via `@vercel/otel`) conforme PRD original. Re-avaliar em Epic 2 se surgir duplicação. |

### Sentry `beforeSend` hook (NFR12)

```typescript
// apps/web/sentry.server.config.ts
import * as Sentry from '@sentry/nextjs'

Sentry.init({
  dsn: process.env.SENTRY_DSN,
  environment: process.env.NODE_ENV,
  beforeSend(event) {
    // Remover PII antes de enviar
    if (event.request?.headers) {
      delete event.request.headers['authorization']
      delete event.request.headers['cookie']
    }
    if (event.user?.email) {
      event.user.email = undefined // NUNCA logar email em Sentry
    }
    return event
  },
  // Replay com máscara de texto (não captura conteúdo de inputs)
  integrations: [Sentry.replayIntegration({ maskAllText: true, blockAllMedia: true })],
  replaysSessionSampleRate: 0,    // desactivado por default
  replaysOnErrorSampleRate: 0.1,  // apenas em erros
})
```
