## User Story

**As a** developer da equipa Expressia,
**I want** um monorepo pnpm funcional com Next.js 15 (App Router), TypeScript strict, ESLint e Vitest configurados e validados,
**so that** a equipa pode começar a implementar features com garantias de qualidade desde o primeiro commit, e o CI/CD da Story 1.2 tenha um repo estável para instrumentar.

---

## Dependências

- **Sem pré-requisitos de stories anteriores** — é a primeira story da Epic 1.
- `packages/db/` já existe com schema Drizzle e migrations. Esta story NÃO toca nesse package — apenas cria a estrutura em volta dele.
- Story 1.2 (CI/CD) depende desta story estar Done.
- Story 1.3 (Supabase/Drizzle bootstrap) depende desta story estar Done.

## Acceptance Criteria

- [ ] **AC1:** Repositório tem estrutura monorepo pnpm workspaces com `package.json` raiz a declarar `packages/*` e `apps/*` como workspaces.
- [ ] **AC2:** `apps/web/` contém scaffold Next.js 15 App Router com TypeScript strict (`strict: true` em `tsconfig.json`) e sem erros de typecheck (`pnpm typecheck` sai com código 0).
- [ ] **AC3:** `apps/web/` arranca em dev (`pnpm --filter web dev`) em < 30 segundos sem erros de consola.
- [ ] **AC4:** ESLint configurado com ruleset que inclui `@typescript-eslint/recommended`, `next/core-web-vitals`; `pnpm lint` sai com código 0 no scaffold inicial.
- [ ] **AC5:** Vitest configurado em `apps/web/` e em `packages/db/`; `pnpm test` corre e sai com código 0 (pode não haver testes reais ainda — o runner deve funcionar sem erros de configuração).
- [ ] **AC6:** Scripts raiz funcionais: `pnpm lint`, `pnpm typecheck`, `pnpm test`, `pnpm build` — cada um corre nos workspaces relevantes via Turbo ou filter pnpm.
- [ ] **AC7:** `packages/db/` existente está correctamente referenciado como workspace e o seu `package.json` está alinhado com a estrutura monorepo (nome `@meu-jarvis/db`, exports corretos).
- [ ] **AC8:** Todos os imports em `apps/web/` são absolutos com alias `@/` (Constitution Article VI — CON5); verificado que nenhum import relativo com `../` existe no código gerado.
- [ ] **AC9:** Ficheiro `.env.example` criado na raiz com variáveis necessárias documentadas (sem valores reais).
- [ ] **AC10:** `apps/web/vercel.json` criado com `regions: ["fra1"]` (Frankfurt) e cron diário em `/api/cron/daily` às 03:00 UTC.

### Convenções de nomenclatura obrigatórias (CLAUDE.md + CON5)

- Componentes: PascalCase (`WorkflowList.tsx`)
- Hooks: prefixo `use` (`useWorkflowOperations`)
- Ficheiros: kebab-case (`workflow-list.tsx`)
- Constantes: SCREAMING_SNAKE_CASE
- **Imports: SEMPRE absolutos** — `import { X } from '@/components/Y'`, NUNCA `../../../`
