### 4. `<DateDisplay>` novo em `packages/ui`

**AC4:** Criar `<DateDisplay>` com `Intl.DateTimeFormat('pt-PT')` + presets standard:

(a) **Criar** `packages/ui/src/components/DateDisplay/DateDisplay.tsx` com:

```typescript
export interface DateDisplayProps {
  /** Data a apresentar — aceita Date, ISO string, ou timestamp ms. */
  readonly value: Date | string | number;
  /**
   * Preset de formatação PT-PT:
   *  - `short`     → "14/03/2026"            (default — Intl numeric)
   *  - `long`      → "14 de março de 2026"   (Intl longo)
   *  - `time`      → "14:32"                 (HH:mm, hour12: false)
   *  - `datetime`  → "14/03/2026 14:32"      (short + time, separador espaço)
   */
  readonly preset?: 'short' | 'long' | 'time' | 'datetime';
  /** Override de className para wrap span (default: vazio). */
  readonly className?: string;
}

export function DateDisplay({ value, preset = 'short', className = '' }: DateDisplayProps): React.ReactElement;
```

(b) **Helper standalone** `formatDate(value, preset?)` exportado para uso em strings (não-JSX) — análogo a `formatEuroCents` em `<MoneyDisplay>`.

(c) **Implementação:** instanciar `Intl.DateTimeFormat('pt-PT', options)` UMA vez por preset (module-level constants, mesmo pattern de `numberFormatter` em MoneyDisplay). Opções:
- `short`: `{ year: 'numeric', month: '2-digit', day: '2-digit' }` (output canónico PT-PT: `14/03/2026`)
- `long`: `{ year: 'numeric', month: 'long', day: 'numeric' }` (output PT-PT: `14 de março de 2026`)
- `time`: `{ hour: '2-digit', minute: '2-digit', hour12: false }` (output: `14:32`)
- `datetime`: composição `short + " " + time` (consistente com `language-standards.md` PT-PT vs `MM/DD/YYYY` americano)

(d) **Tratamento de input:** normalizar `value` para `Date` antes do `format()`. Se `value` for string ISO, usar `new Date(value)`. Se for `number`, idem. Se for `Date`, usar directamente.

(e) **Render JSX:** `<span className={`tabular-nums ${className}`.trim()}>{text}</span>` — `tabular-nums` para alinhamento em colunas (consistente com `<MoneyDisplay>` que também usa `tabular-nums`).

(f) **Barrel** `packages/ui/src/components/DateDisplay/index.ts` re-exporta `DateDisplay`, `DateDisplayProps`, `formatDate`.

(g) **Substituir `formatDate` local em `CardStatementCard.tsx:21`** pelo novo `<DateDisplay>` ou pelo helper `formatDate` exportado (decisão @dev em implementação — função local actual é `iso → DD/MM/YYYY`, exactamente o preset `short`).

**AC4 PASS:** `import { DateDisplay, formatDate } from '@meu-jarvis/ui'` resolve em `apps/web`; vitest unit tests cobrem os 4 presets contra input fixo `new Date('2026-03-14T14:32:00Z')` (cuidado com timezone — usar Date local ou UTC consistentemente, documentar em JSDoc).

### Pattern de import absoluto preservado (NFR19)

Todos os imports cross-package devem ser absolutos (`@meu-jarvis/ui`, `@meu-jarvis/db`) — nunca relativos cross-package (`../../packages/ui/`). Aplicar:

```typescript
// ✓ Correcto (apps/web/src/app/(app)/financas/_components/AccountBalanceCard.tsx)
import { MoneyDisplay } from '@meu-jarvis/ui';

// ✗ Errado (proibido NFR19)
import { MoneyDisplay } from '../../../../packages/ui/src/components/MoneyDisplay';

// ✓ Correcto (packages/ui/src/components/MoneyDisplay/MoneyDisplay.tsx — imports intra-package)

// OU
import { tokens } from '../../tokens';  // relativo intra-package é aceitável
```
