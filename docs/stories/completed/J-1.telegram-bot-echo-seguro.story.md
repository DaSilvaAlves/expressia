## Story

**As a** Eurico (único utilizador do Jarvis v1),
**I want** que o Jarvis tenha um webhook de Telegram seguro que receba as minhas mensagens, verifique que sou eu, e responda com echo,
**so that** a tubagem do canal Telegram fique provada de ponta a ponta — sem tocar no motor — antes de ligar a identidade e as acções (J-2).

## Acceptance Criteria

1. `POST /api/telegram/webhook` aceita updates válidos do Telegram (content-type `application/json`, corpo com `update_id` e `message` ou `callback_query`), devolve `200 OK` e loga o `update_id`.
2. Pedidos sem o cabeçalho `X-Telegram-Bot-Api-Secret-Token` correcto são rejeitados com `401` (resposta vazia, sem stack trace, sem processar o update).
3. Só o `chat_id` do Eurico (configurado via variável de ambiente `TELEGRAM_ALLOWED_CHAT_ID`) é processado; qualquer outro `chat_id` é ignorado silenciosamente (devolve `200` sem resposta ao utilizador, regista log de aviso com o `chat_id` bloqueado).
4. Para mensagens de texto válidas do Eurico, o webhook chama a Bot API `sendMessage` com o texto `"Echo: {texto original}"` de volta ao `chat_id` do Eurico.
5. `TELEGRAM_BOT_TOKEN` (token do bot @BotFather) e `TELEGRAM_WEBHOOK_SECRET` (secret configurado no `setWebhook`) vivem exclusivamente em variáveis de ambiente Vercel (UE) — nunca hardcoded, nunca em git. `TELEGRAM_ALLOWED_CHAT_ID` segue o mesmo padrão.
6. O `sendMessage` é feito via `fetch` nativo (sem dependência nova no `package.json`).
7. A chamada à Bot API em `sendMessage` inclui tratamento de erro (resposta não-2xx → log de erro + retorno `500`); falha de envio não expõe detalhes ao Telegram.
8. `pnpm lint` (--max-warnings=0), `pnpm typecheck`, `pnpm --filter @meu-jarvis/web test` (incluindo novos testes unitários da story), `pnpm build` e `pnpm check:rls` passam sem erros.

### Nota de âmbito (J-1 não precisa de tabela nova)

Esta story é propositadamente echo puro — prova a tubagem sem base de dados. A identidade do Eurico é validada por `TELEGRAM_ALLOWED_CHAT_ID` (env var), não por uma tabela `telegram_link` — essa tabela é criada em J-2, quando se liga ao motor. Logo, **J-1 não cria migrations, não toca no schema Drizzle e não acciona o RLS gate** (o `pnpm check:rls` passa por não haver tabelas novas com `household_id`).

[AUTO-DECISION] Allowlist via env var em J-1 vs tabela `telegram_link` em J-2 → env var. Razão: manter J-1 como prova de tubagem isolada, sem acoplamento à DB. O PRD §4.8 confirma que `telegram_link` é o contrato de identidade de J-2 em diante; forçá-la em J-1 aumentaria o risco sem benefício da espinha.

### Verificação do secret token — timingSafeEqual

A comparação deve ser constante para evitar timing attacks (NFR-J4). Padrão:

```ts
import { timingSafeEqual } from 'node:crypto';

export function verifyWebhookSecret(request: Request): boolean {
  const secret = process.env.TELEGRAM_WEBHOOK_SECRET;
  if (!secret) return false;
  const header = request.headers.get('x-telegram-bot-api-secret-token') ?? '';
  const a = Buffer.from(header);
  const b = Buffer.from(secret);
  if (a.length !== b.length) return false;
  return timingSafeEqual(a, b);
}
```

`timingSafeEqual` lança se os buffers tiverem comprimentos diferentes — proteger com verificação prévia de comprimento.

### sendMessage via fetch nativo

```ts
// apps/web/src/lib/telegram/client.ts
export async function sendMessage({
  chatId,
  text,
}: {
  chatId: number;
  text: string;
}): Promise<void> {
  const token = process.env.TELEGRAM_BOT_TOKEN;
  if (!token) throw new Error('[telegram] TELEGRAM_BOT_TOKEN não definido');
  const res = await fetch(
    `https://api.telegram.org/bot${token}/sendMessage`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ chat_id: chatId, text }),
    }
  );
  if (!res.ok) {
    console.error(`[telegram] sendMessage falhou: ${res.status}`);
    throw new Error(`[telegram] sendMessage falhou: ${res.status}`);
  }
}
```

A Bot API HTTP é JSON simples — sem dependência nova. Confirmado: `apps/web/package.json` não tem cliente Telegram (PRD §8 / `docs/prd-jarvis.md`).
