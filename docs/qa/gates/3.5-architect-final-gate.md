## 9. Lessons Learned (for future stories)

1. **Pre-impl ratify must verify dep availability** — Pre-impl `3.5-architect-dps-ratify.md` recomendou A (Tab Radix) silently sem verificar Radix instalado. D-3.5.2 refinement caught isto. Future ratifies devem incluir explicit dep check para recomendações que mencionem libs específicas.

2. **DST tests pay off pattern** — R3 DST mitigation via tests específicos (29 Mar + 25 Out 2026) é precedent reusable Stories 3.7 (Inngest cron) + 6.x (dashboard Visão).

3. **`useOptimistic` React 19 + DragOverlay portal único = clean drag UX** — Pattern reusable beyond Story 3.5 (Stories 3.7+ se future drag UI).

4. **Helper module pattern (week-helpers.ts) beats inline date-fns calls** — G1.3 refined: concentrar em helper module em vez de só num componente (WeekNavigation) é melhor para testability + reusability.

5. **Story 3.4 pattern `useState<ToastState>` + `<div role="status" aria-live="polite">` é canonical Epic 3** — Sonner NÃO instalado, inline custom é o padrão até Story 3.6+ extract para `useInlineToast()` hook.
