Pode adicionar uma secção opcional "AIOX Framework Tools" no fim para credenciais de tools de desenvolvimento, mas as variáveis de aplicação devem ser as canónicas listadas.

### Nice-to-Have — 2

1. **Risk section missing (point #8 da checklist)** — Story não tem secção dedicada a riscos/mitigations. Para uma scaffold story isto é defensável (riscos triviais), mas idealmente em stories futuras incluir mesmo que mínimo (ex: "incompatibilidade de versões React 19 ↔ libs Vitest", "drizzle-kit conflito com pnpm workspace resolution"). **Não-bloqueante para esta story.**
