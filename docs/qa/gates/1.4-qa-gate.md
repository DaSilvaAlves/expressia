## Issues

### LOW (não-bloqueantes)

#### PERF-001 — Risco marginal de cold pull em CI primeira run
- **Severity:** low
- **Category:** performance
- **Description:** Imagem `pgvector/pgvector:pg16` (~150MB) precisa de pull na primeira run em CI fresco. Local 23s; estimativa CI 30-65s. Margem ampla face ao timeout de 12 min do job mas o budget AC7 (60s) pode ser roçado em ambiente lento.
- **Recommendation:** Não bloquear. Após primeira CI run pós-push, registar tempo real. Se exceder 60s só na suite, considerar pre-pull via `docker pull pgvector/pgvector:pg16 &` num step early ou layer cache via `docker/setup-buildx-action`.
- **Blocking:** false

#### OBS-001 — payment_events inacessível via Supabase Studio com sessão authenticated
- **Severity:** low
- **Category:** docs
- **Description:** Decisão de segurança correcta (apenas service_role acede via webhook Stripe) mas remove debugging via Studio com sessão `authenticated`. O Dex sinalizou na "areas_para_qa_atencao".
- **Recommendation:** Não bloquear. Documentar em runbook Supabase Studio futuro que inspecção de `payment_events` requer service_role key. Se aparecer pedido legítimo de owner/admin SELECT, criar policy dedicada — não relaxar a actual.
- **Blocking:** false
